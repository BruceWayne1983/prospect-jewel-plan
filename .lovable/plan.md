## Problem
Contact details (phone, email, website, address) on prospect profiles are flagged "AI-estimated â€” verify before calling/emailing" because they were generated by the LLM without grounding in real sources. The user requires accurate data, not guesses.

## Plan

### 1. New edge function `enrich-contact-details`
For a single prospect (or retailer), do:

1. **Find official site** â€” if `website` is missing or marked AI-estimated, run Firecrawl `/search` with query `"<name>" <town> UK contact` filtered to non-directory domains; pick the top result whose domain is not yell/facebook/linkedin/google.
2. **Probe the URL** (HEAD then GET) â€” confirm 200 OK; reject if redirects to a holding/parking page (detect via `<title>` keywords like "domain for sale", "coming soon").
3. **Scrape contact page** â€” Firecrawl `/scrape` on `<root>` and `<root>/contact`, format `markdown` + `json` extraction with a strict schema:
   ```json
   { "phone": "string|null", "email": "string|null",
     "address": "string|null", "opening_hours": "string|null" }
   ```
   The JSON extractor uses `prompt: "Extract ONLY contact details that appear verbatim on this page. If a field is not present, return null. Do not infer."`
4. **Cross-check with Google Maps listing** (via Firecrawl search `"<name> <town>" site:google.com/maps`) for phone + address sanity check.
5. **Companies House lookup** â€” call existing `companies-house` function for registered address.

Each returned field carries:
```ts
{ value: string, source: 'website' | 'google_maps' | 'companies_house' | 'manual',
  source_url: string, scraped_at: ISO, confidence: 'high'|'medium'|'low' }
```

Save to `discovered_prospects.verification_data.contact_provenance` (JSONB) and overwrite top-level `phone`/`email`/`website`/`address` only with `confidence: high` results. Set `verification_status = 'web_verified'` (or `'manually_verified'` if user confirmed).

### 3. Bulk verification
Add **"Verify contacts"** bulk action on `ProspectDiscovery.tsx` for selected rows; calls the function in batches of 5 with a 1.5s delay (respects Firecrawl rate limits).

### 4. Block calls/emails on unverified data
- The phone `<a href="tel:">` and email `<a href="mailto:">` links remain clickable but show a confirm dialog if `confidence !== 'high'`: "This contact has not been verified. Continue anyway?"

### 5. Discovery hardening
Update `discover-prospects` to **never** populate `phone`/`email` from the LLM. Only `website` (already URL-probed) and `address` (from the discovered listing) are saved at discovery time. Phone/email come exclusively from the new enrichment step. This eliminates AI-fabricated contact data at the source.

### 6. Manual override
Add inline edit (pencil icon) per field; saving sets `source: 'manual'`, `confidence: 'high'`, and stores Emma's user_id + timestamp.
